## Launching Study-Hub: A Unified Knowledge System for the Modern Student

I’ve always found it frustrating to juggle **Obsidian** for notes, **ChatGPT** for research, and a cluttered **file manager** for my textbooks. Switching tabs breaks focus, especially when you’re deep into an engineering problem.

To solve this, I built **Study-Hub**—a platform that unifies your notes, your library, and an AI that actually knows what's in your files.

### 🚀 The "No-Framework" Challenge
I deliberately built this without high-level frameworks like React or Tailwind. I wanted to master the fundamentals of the web:
* **Frontend:** Vanilla JavaScript and Pure DOM manipulation. Every transition, sidebar toggle, and modal was built from scratch using custom CSS and the Web API.
* **Custom Markdown Engine:** I developed a bespoke engine to handle real-time note rendering without relying on heavy external libraries.
* **Backend:** A modular Node.js & Express architecture designed for scalability.
* **Database:** PostgreSQL for robust, relational data management.

### 🧠 Features that Matter
* **Integrated RAG (Retrieval-Augmented Generation):** Chat directly with your notes and PDFs. I’ve integrated the OpenRouter API so you can query your own data without constantly uploading files to a browser.
* **PDF Streaming:**  Built a custom PDF management and ingestion pipeline that uses pdf-parse under the hood.
* **Deep Organization:** A folder system for both the Note Writer and the PDF Library.
* **Full Security:** Robust authentication handled via JWT (JSON Web Tokens).

### 🛠️ Architecture & Tech Stack
* **Languages:** JavaScript (ESM), HTML5, CSS3
* **Environment:** Node.js, Express.js
* **Persistence:** PostgreSQL
* **AI:** RAG system via OpenRouter
* **Hosting:** Deployed on Render

This project was my deep dive into building production-ready multimodal full-stack applications. It’s not just a demo; it’s the tool that can be actually used for studies.
